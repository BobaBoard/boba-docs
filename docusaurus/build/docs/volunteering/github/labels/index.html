Issue Labels

BobaBoard uses labels to classify issues and feature requests according to a variety of criteria.

For ease of reference, labels are further divided in the following macro-categories:

  • 🎨 Thematic labels indicate the overall theme of the issue or feature request.
  • 🛠 Skills labels indicate the technical skills necessary to tackle the issue or feature request.
  • 📦 Repository labels indicate the repositories the issue or feature request touches upon.
  • ⏩ Stage labels indicate the next step in the issue-tackling process.
  • 💪 Work-type labels indicate special characteristic of the work surrounding the issue.
  • 🔥 Priority labels indicate the priority of the issue or feature request.
  • 🌟 Special labels indicate the issue meets certain special criteria.

We also have a set of other labels that do not fit in these categories.

tip

Click on any label to see the associated issues.

Thematic labels​

🎨 Accessibility
Let's make BobaBoard more accessible!
🎨 Community
Tools to improve BobaBoard's communities experience.
🎨 Customization
Personalizing BobaBoard for personal or community use.
🎨 Decentralization
Look, ma! No NFTs.
🎨 Frustration busters
Make BobaBoard a pleasure to use
🎨 Integrations
Anything related to the ability to exchange data with 3rd party software.
🎨 Moderation
Anything to do with content moderation, both community and personal.
🎨 Privacy
Ensure everyone's privacy is respected.

Skills labels​

🛠 DevOps
🛠 Documentation (reading)
Requires the ability to read and understand (sometimes poorly written) documentation
🛠 JavaScript
This task requires an understanding of JavaScript
🛠 WebDev
Requires an understanding of broad web development technologies

Repository labels​

📦 meta
Meta issues about improving BobaBoard community & processes
📦 boba-backend
An issue that involves the boba-backend repository
📦 boba-components
An issue that involves the boba-components repository
📦 boba-editor
An issue that involves the boba-editor repository
📦 boba-frontend
An issue that involves the boba-frontend repository

Stage labels​

⏩ stage: awaiting reproduction
⏩ stage: blocked
⏩ stage: design
Open issues need deliberation before implementation can proceed
⏩ stage: ready to implement
⏩ stage: UX design
The issue is currently undergoing interface/interaction desing.

Work-type labels​

💪 self-contained
The task doesn't have broad repercussions on the BobaBoard codebase
💪 volunteer-friendly
This tasks is approachable to volunteers!

Priority labels​

🔥P4
Priority NONE: No current plans to do this, but still considering it
🔥🔥P3
Priority LOW: This can wait until we have time
🔥🔥🔥P2
Priority MEDIUM: This can wait a few releases
🔥🔥🔥🔥P1
Priority HIGH: Do this for the next release
🔥🔥🔥🔥🔥P0
Priority CRITICAL: Drop everything to work on this

Special labels​

🌟 Ms. Boba's Fave
A task Ms. Boba would *love* to get done
